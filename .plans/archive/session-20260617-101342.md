## 会话复盘

### 发生了什么
- 修复 Codex provider 配额配置从绝对 token 转为官方 usage 百分比阈值，并保留旧 token 字段兼容。
- 用量查询页新增官方未返回 token usage 时的明确提示，避免把“官方没给数据”误判成“没有查询”。
- 本地容器已用新镜像重建，运行容器不注入代理环境变量。

### 高信号问题
- Docker build 网络依赖: 构建 tls-sidecar 的 Go 依赖时，直连 `proxy.golang.org` 可能连接被拒绝；使用构建期 `HTTP_PROXY/HTTPS_PROXY=http://host.docker.internal:7890` 可通过，但运行容器仍应只使用配置管理里的 provider proxy，不把代理 env 注入 runtime。

### 下次默认做法
- 重建本地 aiclient2api 镜像时，先直接 build；若 Go/NPM 依赖下载失败，只对 build 阶段加本机代理 build-arg，并在容器启动后确认 `.Config.Env` 不包含 `HTTP_PROXY/HTTPS_PROXY`。

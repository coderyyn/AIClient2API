详细内容怎么查

`request-audit` 默认适合长期保留安全明细；`ai-monitor` 适合短时排查协议转换链路，但当前日志会经过脱敏，不是完整原始 prompt 归档。

如果要临时保存原始 prompt，应单独实现显式危险开关、短保留期、访问隔离和清理确认，不建议混进默认审计。

原始请求采集

REQUEST_AUDIT_RAW_CAPTURE：只对白名单 key 临时保存原始 prompt，用于判断公共前缀、截断和结构变化。

未读取

默认关闭。开启后会保存原始 prompt 和转换后的请求体；建议只开 10-30 分钟，分析完立即关闭。

风险提示：会保存原始 prompt。文件默认写入 configs/request-audit-raw，并由 TTL 定时清理；不要长期打开。

什么时候开启

只在需要解释“多次请求缓存率为什么低”时短时开启，例如怀疑公共前缀变短、工具定义变化、附件导致截断，或账号/模型切换影响缓存。

在哪里查看

采集文件写入 configs/request-audit-raw。页面保留审计汇总，原始 prompt 不在默认表格中展示，避免误泄露。

怎么收尾

分析完成后选择“关闭原始请求采集”并保存。TTL 会定时清理过期文件，但不应把它当作长期日志归档。

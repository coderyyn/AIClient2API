新增能力设计

Codex 账号 token 统计

在 model-usage-stats 中新增账号维度，键为 openai-codex-oauth:uuid。保留现有 provider/model/daily 聚合，新增 accounts、daily[date].models 和 daily[date].accounts。

入口 用量查询页查看 Codex 账号。

Codex token 配额路由

在 provider pool 选择前过滤超额账号。字段只对 Codex 生效：codexMax5hTokens 和 codexMaxWeeklyTokens。超额账号不可选，所有账号超额时返回明确 429。

依据 只看 token，不看请求次数。

早晨 5h 窗口预热

新增 Codex prewarm 服务，每天按配置时间遍历所有启用 Codex 账号，做低成本请求以提前触发 5h 窗口。按账号、日期、时间点记录执行状态，避免服务重启后重复触发同一窗口。

默认 06:30 与 11:30 Asia/Shanghai，每账号每窗口 2 次。

官方 Codex /usage 接入

复用 CodexApiService 的鉴权信息，只做只读 usage 查询。格式化层已兼容多种 daily/weekly/total token 字段形态；接口不可用或字段变化时保留原 rate limit 百分比展示。

已完成 官方 usage 查询默认缓存 1h；现有用量页刷新按钮和 refresh=true 可绕过缓存。

API Potluck key 用量视图

在插件管理端增加 key 维度的每日、本周和累计用量，不要求用户端输入 key 才能看。管理端可直接查看分发出去的 key 的每日 token、累计 token、缓存命中率和请求次数。

入口 API Potluck 插件页。

排序与筛选

API Potluck 管理端默认按今日 Token 排序，提供本周 Token、累计 Token、调用次数、QPS/RPM 等下拉选项。

已完成 token 优先。

这套设计的核心是同一份 token 事实源服务多个入口：Codex 账号用于路由和容量判断，API Potluck key 用于分发后审计，两者不要混成一个概念。

缓存命中率排查项

新增排查主题：美国号池今天额度消耗明显变快，但 total token 与之前接近甚至更少，优先验证是否是 prompt cache 命中率下降导致实际消耗升高。

核心指标

按 Codex 账号、模型、日期展示 cachedTokens、promptTokens、totalTokens 和 cacheHitRatio。

公式 cachedTokens / promptTokens

排查假设

最近改动可能改变转换后的 prompt、tools、metadata、message 顺序或路由分散度，导致原本 50% 以上的缓存命中下降。

方案联动

现有统计无法持久回答“账号 + 模型 + 日期”的命中率，需要把 cache hit ratio、账号维度和模型日期维度加入定制分支。

远端验证项 结果 结论
2026-06-16 每日总命中率 15.18% 显著低于 6 月初多日 50%+ 基线，用户观察成立。
当前容器日志样本 gpt-5.5: 18.01%，gpt-5.4: 90.33% 不是所有模型都低，主要由 gpt-5.5 拉低。
账号维度样本 8dec7a3e: 11.13%，b4a54e62: 36.73% 一个账号显著更低；账号只以哈希展示，不暴露真实账号。
路由分散假设 当前样本只看到 2 个账号承担流量 不像是大量账号随机分散导致，但账号切换可能加剧下降。
详细排查方案已持久化到 docs/yyn-codex-usage-cache-hit-investigation.md。远端验证确认今天低命中，方案更新为：必须加入账号+模型+日期维度的 cache hit ratio；路由策略先做观测和告警，不默认自动按命中率调度。

数据流

用量统计要从请求处理链路拿到真实选中的 provider uuid，否则只能按 provider 类型统计，无法回答“某个 Codex 账号今天用了多少 token”。

请求进入
  -> provider pool 选择 Codex 账号
  -> common.js / api-manager.js hook 透传 providerUuid 与 providerName
  -> model-usage-stats finalizeRequest 写入账号维度 daily/weekly/total
  -> provider-pool-manager 读取 token 窗口统计，过滤超额账号
  -> /api/usage 展示 Codex 账号用量
  -> api-potluck 插件页展示分发 key 的每日与累计用量

本地统计窗口

5h 使用滚动窗口 token 统计，服务 provider 选择过滤；每日、每周、累计用于 UI 展示和诊断。

官方 usage 数据

只读获取 Codex 账号官方 daily/weekly/total；字段稳定后做缓存，避免每次打开页面都打官方接口。
